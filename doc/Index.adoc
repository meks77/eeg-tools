# EEG Tools

## Erscheinungsbild

Die Applikation soll eine Webapplikation sein. Sie soll optimiert für diverse Bildschirmgrößen sein. Darunter fallen Handy, Tablets, Laptops und größere Monitore.

Das Layout bzw. Design soll so gestaltet sein, dass es zur Homepage der Hill EEG(https://www.hill-stromeeg.at) passt.

## Funktionalitäten

### Einfache Berechnung der Ersparnis für Verbraucher

Die Benutzer:in hat eine einfache Eingabemöglichkeit für

* den jährlichen Energiebedarfs
* den Arbeitspreis pro kw/h inkl. Steuer beim aktuellen Stromlieferanten
* die typischen Anwesenheitszeiten Zuhause(Ganztags, Vormittags, Nachmittags, Abends)

Aufgrund der Anwesenheitszeiten soll die prozentuelle Abdeckung mit EEG-Strom automatisch angepasst werden. Die prozentuelle Abdeckung je nach Anwesenheit, wird statistisch mit aufgrund von bestehenden Benutzern erhoben, und für die Berechnungsbasis bereitgestellt.

Die Benutzer:in hat die Möglichkeit die prozentuele Abdeckung direkt einzugeben, und die Vorbelegung dadurch zu übersteuern.

Mit diesen Eingaben soll die Ersparnis berechnet werden, welche durch den Bezug des Strom aus der EEG erzielt wird. Dabei ist zu berücksichtigen dass auch eine Mitgliedsgebühr von 10 Euro, und eine Einschreibgebühr von 15 Euro anfällt. Daher wäre es sinnvoll die Ersparnis im ersten und im zweiten Jahr auszuweisen. Der Preis für die kw/h bei der EEG ist aktuell 10 Cent. Es soll außerdem auch noch die Ersparnis der Netzgebühr und sonstiger Abgaben berücksichtigt werden, welche bei einer regionalen EEG eingespart werden können. Die Kosten bei der EEG sollen einzeln (z.B. Netzgebühr, Ökostromabgabe, etc.) aufgelistet werden, und den Kosten bei dem aktuellen Stromanbieter gegenüber gestellt werden. Des weiteren sollen die Ersparnisse bzw. Mehrausgaben angeführt werden.

## Technik

Die Applikation ist eine serverless Webanwendung, die auf GitHub Pages gehostet wird.

* **Sprache:** Kotlin
* **Framework:** [Compose HTML](https://github.com/JetBrains/compose-jb) (Kotlin Multiplatform)
* **Styling:** Tailwind CSS (für Responsive Design)
* **Build-Tool:** Gradle
* **Hosting:** GitHub Pages

### Testkonzept

Die Applikation wird auf verschiedenen Ebenen automatisiert betestet:

1. **Fachlichkeit (Logic):**
   * **Technologie:** [Kotest](https://kotest.io/)
   * **Methodik:** Specification By Example (Given/When/Then)
   * **Scope:** Schnelle Tests der Berechnungslogik in `commonMain`.

2. **UI & Responsive Design:**
   * **Technologie:** [Playwright](https://playwright.dev/)
   * **Scope:** E2E-Tests in verschiedenen Browsern (Chromium, Webkit) und Viewports (Mobile, Desktop).
   * **Layout-Tests:** Automatisierte Checks für unterschiedliche Bildschirmgrößen.

### Infrastruktur

* **Repository:** GitHub
* **CI/CD:** GitHub Actions (`.github/workflows/deploy.yml`)
* **Deployment:** Automatisches Deployment auf GitHub Pages bei jedem Push auf den `main`-Branch.
